## 系统总览

```
电路评估系统
├── 流水线 A (图像输入): YOLO检测 → CRNN OCR → 接线 → LLM评价 → 标注图
├── 流水线 B (文本输入): LLM结构化 → 安全验证 → SPICE仿真 → 质量评分
└── 模型: YOLOv8n(CGHD/HCD) + CRNN(CNN+BiLSTM+CTC)
```

### CRNN OCR 模型

| 属性 | 值 |
|------|-----|
| 架构 | CNN特征提取 + BiLSTM + CTC Loss |
| 输入 | 灰度图, 32×128 px (等比缩放+填充) |
| 预处理 | 反色 (255-img), 归一化到 [-1, 1] |

**CNN 骨干 (5层)**:

| 层 | 类型 | 输入通道 | 输出通道 | 池化 |
|----|------|---------|---------|------|
| 1 | Conv3x3+BN+ReLU | 1 | 64 | MaxPool2d(2,2) |
| 2 | Conv3x3+BN+ReLU | 64 | 128 | MaxPool2d(2,2) |
| 3 | Conv3x3+BN+ReLU | 128 | 256 | — |
| 4 | Conv3x3+BN+ReLU | 256 | 256 | MaxPool2d((2,1)) |
| 5 | Conv3x3+BN+ReLU | 256 | 512 | MaxPool2d((2,1)) |

CNN 输出维度: (Batch, 512, 2, W/4) → 重塑为 (Batch, W/4, 1024)

**序列模型**:
- 2层双向 LSTM, hidden_size=256
- Dropout=0.2
- 输出: (Batch, W/4, 512)

**输出层**: Linear(512 → 37), CTC Loss (blank=0)

**训练参数**:

| 参数 | 值 |
|------|-----|
| Epochs | 20 |
| Batch | 64 |
| LR | 0.0005 |
| 优化器 | AdamW (weight_decay=1e-4) |
| 调度器 | CosineAnnealingLR |
| 数据增强 | 亮度抖动 (30%), 高斯噪声 (20%) |
| 字符集 | `0123456789.kKmMΩμunpFV AHz-+` (28字符) |
| 类别数 | 29 (28 + CTC blank) |
| Best Val Loss | 0.084 |
| 权重路径 | `runs/ocr_crnn/best.pt` |

**推理**: CTC 贪婪解码 (非束搜索)

## 检测流水线

### 主脚本: `src/vision/unified_pipeline.py`

```
输入图片
    │
    ├── Step 1: YOLOv8 CGHD 56类 元件检测
    │   ├── 过滤: 跳过 text/junction/crossover/terminal/probe (CGHD非元件类)
    │   ├── 阈值: 元件 conf≥0.40, 节点 conf≥0.20
    │   ├── CGHD→HCD 类名映射 (56类→25类)
    │   ├── NMS去重: 同族重叠检测合并 (diode/LED/Zener, V-DC/V-AC等)
    │   └── 元件 bbox + 类别 + 置信度
    │
    ├── Step 2: HCD circuit_text 模型 文字区域检测
    │   ├── 过滤文字在元件框内的误检 (重叠>50%即排除)
    │   └── 文字裁剪图
    │
    ├── Step 3: CRNN OCR 文字识别
    │   ├── 预处理: 等比缩放至32高, 填充至128宽, 反色归一化
    │   ├── CTC贪婪解码
    │   ├── _clean_ocr(): 简写展开 (3V3→3.3V, 4R7→4.7Ω)
    │   ├── _ocr_variants(): 生成候选解读
    │   │   ├── 结尾"H" → 尝试 kΩ/MΩ/Ω (CRNN混淆修正)
    │   │   ├── 结尾"V" → 尝试 Ω/μF/F (Ω↔V 混淆修正)
    │   │   └── 双位数 → 插入小数点变体 (22k→2.2k)
    │   └── 改正记忆: 加载 ocr_corrections.json
    │
    ├── Step 4: 数值匹配 (proximity-first)
    │   ├── 距离加权: 曼哈顿距离 ≤250px
    │   ├── 模式验证: VALUE_PATTERNS (电阻Ω, 电容F, 电感H, 电压V, 电流A)
    │   ├── 反模式过滤: ANTI_PATTERNS (二极管不能有Ω/F/H值)
    │   ├── 第一轮: 最短距离精确匹配
    │   ├── 第二轮: 候选变体填充未匹配元件
    │   └── 高置信修正: conf>0.6的元件强制修正单位
    │
    ├── Step 5: 设计位号分配
    │   └── R1,R2,... C1,C2,... D1,D2,... LED1,... Q1,Q2,... U1,U2,...
    │
    ├── Step 6: 端口计算 (Sobel方向检测)
    │   ├── PORT_POSITIONS 字典 (归一化坐标, 基于CGHD classes_ports.json)
    │   ├── Sobel边缘分析: 判断元件实际朝向 (水平/垂直)
    │   ├── 旋转: 默认横向, 如Sobel检测为竖向则交换x↔y
    │   └── 转为绝对像素坐标
    │
    ├── Step 7: 接线推理
    │   ├── 节点处理: 网格捕捉(5px) → 并查集合并(8px)
    │   ├── 端口→节点: 250px半径, 路径不穿过其他元件框
    │   ├── 端口→端口: 50px直连 (无节点时回退)
    │   └── 节点→节点: 对齐(25px)或近邻(60px)自动连接
    │
    ├── Step 8: Manhattan布线
    │   ├── L型: 先水平后垂直, 或先垂直后水平
    │   ├── 避障: 线段不与任何元件bbox相交 (Cohen-Sutherland)
    │   └── 失败则跳过
    │
    ├── Step 9: 连接图构建
    │   ├── 并查集合并端口 (通过共享节点)
    │   └── 提取连通组 + 端口标签 (R1.1, LED1.A, V1.+ 等)
    │
    ├── Step 10: LLM评价
    │   ├── 构建中文提示: 元件清单 + 值 + 连接关系
    │   ├── 发送 DeepSeek API (带重试+英文备选)
    │   └── 评价: 电路类型/数值合规/设计缺陷/质量评分
    │
    ├── Step 11: 绘制标注图
    │   ├── 元件: 色框 (按类型) + 设计位号=值
    │   ├── 端口: 红点 + 极性标签 (A/K/+/-/GND/B/C/E)
    │   ├── 节点: 绿点
    │   ├── 连线: 蓝线 (Manhattan路由)
    │   └── OCR: 橙色虚线框 + 原文
    │
    └── Step 12: 保存结果
        ├── _v2_wired.jpg (标注图)
        └── _v2_result.txt (结构报告 + LLM评价)
```

## LLM 集成

### 配置 (`src/llm.py`)

| 参数 | 值 |
|------|-----|
| 默认模型 | DeepSeek V4 Pro |
| API端点 | `https://api.deepseek.com/v1` |
| SDK | OpenAI Python SDK (兼容模式) |
| 备选模型 | Claude Sonnet 4.6 (需 Anthropic API Key) |
| 检测方式 | API Key 前缀: `sk-ant*` → Anthropic, 其他 → DeepSeek |
| 温度 | 0.0 |
| Max Tokens | 2048 |
| 环境变量 | `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `ANTHROPIC_AUTH_TOKEN` |

### 重试机制

因 DeepSeek 对大电路中文提示可能触发内容过滤:
1. 首次: 完整中文提示
2. 重试1: 相同中文提示 (过滤器非确定性)
3. 重试2: 英文摘要 + System Message 保持中文回复
4. 最大重试: 3次

## SPICE 仿真

### 第3阶段 (`src/phase3_spice/`)

**网表生成** (`netlist.py`):
- 支持元件: R, C, L, Diode(含.MODEL), LED(简化Vf模型), BJT, MOSFET, Op-Amp(VCVS)
- 生成 `.cir` 临时文件

**仿真执行** (`simulate.py`):
- 调用 `ngspice` (30秒超时)
- 搜索多个Windows安装路径
- LLM解释仿真结果

### 第4阶段 (`src/phase4_netlist/`)

- LLM直接生成SPICE网表 → 确定性解析 → 无JSON幻觉风险
- 支持实际ngspice仿真回退

## 文本评估流水线

### 评分公式

```
总分 = 100 - (致命错误×30) - (主要问题×15) - (次要问题×5) - (质量问题×3)
```

### 检查项

| 类别 | 检查内容 | 方法 |
|------|---------|------|
| 安全 | 电源对地短路、极性反接、悬空节点 | 确定性规则 |
| 正确性 | 缺元件、拓扑错误、数值不合理 | LLM |
| 质量 | 参数次优、功耗、鲁棒性 | LLM |

## 关键数据结构

### PORT_POSITIONS (归一化坐标, bbox内0~1)

| 元件类型 | 端口坐标 | 说明 |
|---------|---------|------|
| Resistor/Capacitor/Inductor/Diode/LED | (0,0.5), (1,0.5) | 左右两端 |
| V-DC/V-AC/I-DC/I-AC | (0.5,1.0), (0.5,0.0) | 下负上正 |
| GND | (0.5,0.0) | 单端向上 |
| BJT-NPN/PNP | (0,0.5), (0.7,1.0), (0.7,0.0) | B, C, E |
| MOSFET-N/P | (0,0.5), (0.5,1.0), (0.5,0.0) | G, S, D |
| Op-Amp | (0,0.5), (0,0.3), (1,0.5) | IN-, IN+, OUT |

### PORT_LABELS (极性/端子标签)

| 元件类型 | 端口标签 |
|---------|---------|
| 无源元件 (R/C/L) | 1, 2 |
| Diode/LED/Zener | A(阳极), K(阴极) |
| V-DC/I-DC | -(负极), +(正极) |
| V-AC/I-AC | ~, ~ |
| GND | GND |
| BJT | B(基极), C(集电极), E(发射极) |
| MOSFET | G(栅极), S(源极), D(漏极) |
| Op-Amp | -(反相), +(同相), OUT(输出) |
