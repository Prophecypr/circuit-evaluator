# 电路图智能评价系统 — 使用指南

## 零基础快速上手

### 这个项目是做什么的？

你用文字描述一张电路图（有什么元件、怎么连接的），系统会自动：

1. **评分**：给电路打 0-100 分
2. **找错**：找出短路、断路、元件接反等致命错误
3. **诊断**：指出哪里不合理，给出修改建议
4. **仿真**（可选）：用 SPICE 引擎真实计算电路的电压电流

### 准备工作（只需做一次）

#### 第一步：安装 Python

1. 打开 https://www.python.org/downloads/
2. 点击黄色按钮下载，运行安装程序
3. **重要**：安装时勾选 "Add Python to PATH"
4. 完成后打开"命令提示符"，输入 `python --version`，看到版本号说明安装成功

#### 第二步：获取代码

```bash
# 方式A：直接从 GitHub 下载
# 浏览器打开 https://github.com/Prophecypr/circuit-evaluator
# 点击绿色 Code 按钮 → Download ZIP → 解压到任意目录

# 方式B：命令行克隆（需要安装 git）
git clone https://github.com/Prophecypr/circuit-evaluator.git
```

#### 第三步：安装依赖

打开"命令提示符"（Win+R 输入 cmd），进入项目目录：

```bash
cd E:\prophecypr_project\github\circuit-evaluator
pip install -r requirements.txt
```

#### 第四步：设置 API Key

你需要一个 Anthropic API Key（用于调用 Claude 大模型）。

1. 访问 https://console.anthropic.com/ 注册并获取 API Key
2. 在命令提示符中设置（每次打开新窗口都要设置）：

```bash
set ANTHROPIC_API_KEY=sk-ant-你的密钥
```

> 也可以把密钥告诉 Claude Code，让它帮你写入环境变量。

#### 第五步（可选）：安装 Ngspice 仿真引擎

如果想获得精确的电压电流数据，安装 SPICE 仿真器：

1. 打开 https://sourceforge.net/projects/ngspice/files/ng-spice-rework/43/ngspice-43_64.zip/download
2. 下载后解压到 `E:\ngspice`
3. 确认 `E:\ngspice\bin\ngspice.exe` 存在

## 三种使用方式（从简单到强大）

### 方式一：端到端评价（最简单，推荐入门）

一键评价 `data/samples/` 目录下所有电路：

```bash
python -m src.phase1_verify.evaluate
```

输出示例：
```
============================================================
文件: circuit_01_good.txt
总分: 95/100
摘要: 电路基本正确，无致命错误，功能可正常实现。
质量问题 (1):
  - [suboptimal_value] 220Ω电阻导致LED电流约13.6mA，略低于额定20mA。
```

**评价单个电路**：

```bash
python -c "from src.phase1_verify.evaluate import evaluate_circuit; from pathlib import Path; import json; desc = Path('你的电路文件.txt').read_text(encoding='utf-8'); r = evaluate_circuit(desc); print(f'评分:{r[\"overall_score\"]}/100'); print(r['summary'])"
```

### 方式二：SPICE 网表评价（更精确）

先让 LLM 把电路描述转成标准 SPICE 网表，再评价：

```bash
python -m src.phase4_netlist.evaluate data/samples/circuit_01_good.txt
```

输出会显示生成的网表 + 解析结果 + 评分报告。

### 方式三：真实仿真评价（最强大，需要安装 Ngspice）

```bash
python -c "
from pathlib import Path
from src.phase4_netlist.evaluate import evaluate_with_simulation

desc = Path('data/samples/circuit_01_good.txt').read_text(encoding='utf-8')
result = evaluate_with_simulation(desc)

print(f'评分: {result[\"overall_score\"]}/100')
print(f'是否使用了真实仿真: {result.get(\"_simulation_used\", False)}')
print(f'摘要: {result[\"summary\"]}')
"
```

有 Ngspice 时自动仿真，没有时自动回退到纯 LLM 评价。

## 如何添加你自己的电路

在 `data/samples/` 目录下新建一个 `.txt` 文件，按以下格式写：

```
电路描述：
<简单描述这是什么电路>

元器件：
- V1：5V直流电源
- R1：220Ω电阻
- LED1：红色发光二极管（正向压降约2.0V，额定电流20mA）

连接关系：
- 电源V1正极连接到电阻R1的一端
- 电阻R1的另一端连接到LED1的正极（阳极）
- LED1的负极（阴极）连接到电源V1的负极（GND）

预期功能：5V电源通过220Ω电阻限流后驱动LED发光
```

**写作要点**：
- 每个元件要有编号（R1, C2, Q1, LED1...）
- 明确写出参数值（阻值、容值、电压等）
- 连接关系逐条写清楚
- 极性元件（LED、二极管、电解电容）标注正负
- 写明"预期功能"帮助 LLM 理解设计意图

## 理解评价报告

评价报告分四个部分：

| 部分 | 含义 | 示例 |
|------|------|------|
| **总分 (overall_score)** | 0-100 分，越高越好 | 95 = 优秀，20 = 有致命错误 |
| **致命错误 (fatal_errors)** | 会导致电路无法工作或损坏的问题 | 短路、断路、电源反接 |
| **正确性问题 (correctness_issues)** | 电路逻辑是否正确 | 缺少必要元件、参数严重不合理 |
| **质量问题 (quality_issues)** | 可以优化的地方 | 电阻值可以微调、功耗可优化 |

**评分参考**：
- 90-100：电路设计正确，质量良好
- 70-89：基本正确，有优化空间
- 40-69：存在较严重问题，需要修改
- 0-39：有致命错误，电路无法工作

## 常见问题

### Q: 运行报错 "ANTHROPIC_API_KEY environment variable not set"
**A:** 没有设置 API Key。在命令行执行 `set ANTHROPIC_API_KEY=你的密钥`，然后重新运行。

### Q: 评价结果不稳定，同一电路两次评分不一样？
**A:** LLM 有一定随机性，评分浮动 ±10 分是正常的。如果要稳定结果，可以降低模型温度参数（修改 `src/llm.py`）。

### Q: 支持哪些类型的电路？
**A:** 当前测试覆盖了 LED 驱动、三极管放大、RC 滤波、分压、运放电路。更复杂的电路（如开关电源、数字逻辑）可能需要更详细的描述。

### Q: 为什么 Phase 2 评分有时不准确？
**A:** Phase 2 的"文字→JSON结构化"转换环节可能出错，导致后续评价失真。建议使用 Phase 1（端到端）或 Phase 4（网表优先）。

### Q: Ngspice 仿真失败了怎么办？
**A:** 系统会自动回退到纯 LLM 评价模式。仿真失败通常是因为 LLM 生成的网表语法有问题（如三极管引脚顺序），不影响整体评价。

### Q: 可以批量评价很多电路吗？
**A:** 可以。把所有电路 `.txt` 文件放到 `data/samples/`，运行 Phase 1 会自动处理全部：

```bash
python -m src.phase1_verify.evaluate
```
