## 项目概况

### 项目目标

构建**手绘电路图自动识别→接线→评价**的端到端系统，以**骨架引导自适应接线算法**为核心创新，发表 EI / SCI 期刊论文。

### 系统架构

```
图片输入 → YOLO 61类检测(CGHD mAP50=74%) → CRNN OCR(28字符) 
→ 数值匹配 → 骨架引导接线算法 → LLM多元评价 → 标注图+TXT报告
```

## 二、创新点定位

### 核心创新：骨架引导的自适应手绘电路接线算法

```
Zhang-Suen细化 → 骨架路径验证 → 度约束过滤 → 最近邻过滤 → 非对齐补偿(LOS)
```

**解决的问题**：手绘电路图中元件位置误差大（±15px）、线条粗细不一、连接点不精确，传统规则方法（最近邻、纯网格对齐）导致过合并和漏连。骨架引导在保证连通性的前提下，通过度约束和最近邻过滤控制连接密度，避免全连接。

### 系统创新：端到端流水线

YOLO 61类 → CRNN手写OCR → 接线 → LLM多元评价 → 标注图输出

### 应用创新：多LLM电路评审可信度分析

DeepSeek / Claude / Ollama 与人工专家评分的 Spearman 相关性

## 四、阶段二：实验（第3-5周）

### 第3周：接线准确率实验

| 实验 | 内容 | 指标 |
|------|------|------|
| **Full Pipeline** | 完整骨架引导接线算法跑50张 | 4项指标 |
| **Baseline A** | 纯P2J + JJ对齐（无骨架/度约束/最近邻/LOS） | 同上 |
| **Baseline B** | 最近邻端口连接（距离最近两两相连） | 同上 |
| **Baseline C** | MST最小生成树连接 | 同上 |

**四项核心指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| 端口正确连接率 P_recall | TP连接数 / GT总连接数 | 越高越好（召回） |
| 误连率 P_fp | FP连接数 / 算法输出总连接数 | 越低越好（精确） |
| 漏连率 P_fn | FN连接数 / GT总连接数 | 越低越好 |
| 组准确率 G_acc | 完全正确的连通组数 / GT总连通组数 | 越高越好 |

### 第4周：消融实验 + 画风鲁棒性

**5项消融实验**（每次去掉一个模块，其余保持）：

| 消融项 | 去掉的功能 | 验证目标 |
|--------|-----------|---------|
| no_skeleton | 骨架路径验证 | 骨架对路径准确性的贡献 |
| no_degree | 度约束过滤 | 度约束防止过合并的贡献 |
| no_nn | 最近邻过滤 | 最近邻对去冗余的贡献 |
| no_skel_jj | JJ间的骨架验证 | JJ骨架验证的价值 |
| no_sobel | 端口朝向检测 | 朝向对正确连接的贡献 |

**画风鲁棒性**：

| 分析 | 方法 |
|------|------|
| D1 vs D2 分组对比 | 两子集分别统计4项指标 |
| 同电路不同绘制者 | C2_D1 vs C2_D2, C130_D1 vs C130_D2 等对比 |
| 低置信检测影响 | 按YOLO置信度分组分析 |

### 第5周：LLM可信度实验

| 步骤 | 内容 |
|------|------|
| 选图 | 15-20张代表性图片（覆盖简单到复杂） |
| LLM评价 | 3个LLM各对每张图生成评价报告 |
| 人工评分 | 1-2位评分者对每张图打1-5分 |
| 统计 | Spearman相关系数 + 错误分类统计（严格/宽松/严重） |

**第5周末产物**：全部实验数据（CSV格式）+ 对比表格 + 柱状图

## 十一、GT文件格式规范

```
G1: R1.1, C2.1
G2: R1.2, Q1.B, R3.1
G3: C2.2, Q1.C, V1.+
...
```

- `G1, G2...` = 连通组编号
- `R1.1` = 元件designator.端口label
- 每组至少2个端点（单端GND可单独成组）
- 按组编号升序排列
